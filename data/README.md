# Data Directory

## spx_options_sample.csv

**Format** (CSV, comma-separated, header on row 1):

| Column         | Type   | Description                                              |
|----------------|--------|----------------------------------------------------------|
| `strike`       | float  | Strike price K (same currency as spot)                   |
| `expiry_years` | float  | Time to expiry T in fractional years (e.g., 0.25 = 3M)  |
| `market_price` | float  | Mid-market option price `(bid + ask) / 2`               |
| `market_iv`    | float  | Black-Scholes implied volatility from mid price (decimal)|
| `bid`          | float  | Bid price                                                |
| `ask`          | float  | Ask price                                                |
| `open_interest`| int    | Open interest (contracts)                                |
| `type`         | int    | `0` = call, `1` = put                                    |

## Data Source

Generated by `python/fetch_data.py`.

**Real market data:** Uses `yfinance` to fetch the SPY option chain.
Filter criteria applied:
- Open interest > 500
- Bid > 0
- Moneyness 0.80 < K/S < 1.20
- Spread/mid < 10%

**Synthetic data (fallback):** If `yfinance` fails, synthetic data is
generated via the Heston model FFT pricer using:

```
v0=0.04, kappa=1.5, theta=0.04, xi=0.3, rho=-0.7
```

This gives a known ground truth that the calibration engine should recover
within ±5% parameter tolerance.

## Notes

- All implied vols are annualised (fractional, not percentage)
- Prices are in dollars per unit notional (SPY ≈ $500)
- The C++ engine reads this file via `main.cpp`; column order must match
